# Crash

## UV/PV Crash 率

UV Crash 率 = Crash UV / DAU。

## 启动 Crash 率

最严重的 Crash。

## 增量、存量 Crash 率

优先解决增量，持续跟进存量。

## Crash 相关指标

万分之四是优秀的水位。

## 关键问题

 1.尽可能还原 Crash 现场

 - 归因堆栈、设备、App 版本、OS 版本、进程、线程、前后台、内存信息、行为日志等信息。

2.完善告警配置

3.责任归属
- 专项小组轮值
- 制定响应规范

# 业务高可用

1. 数据采集
   - 梳理项目主流程、核心路径、关键节点。
   - Aop 自动采集，统一上报。

2. 告警策略
   - 阈值报警、趋势告警。
   - 特定指标告警，直接上报。

 3. 异常监控
   - Catch 代码块。
   - 异常逻辑。

  4. 兜底策略
   - 配置中心，功能开关。
   - 跳转分发中心。

# 移动端容灾方案

1. 增加功能开关
   配置中心，服务端下发配置控制
   针对场景: 新增功能或改动
2. 统一跳转中心
   界面切换通过路由，路由决定是否重定向。
3. 动态化修复
   1. 热修复能力。
   2. 推拉结合、多场景调用保障到达率。
   3. 跨端容器动态下发 Bundle。
4. 安全模式
   1. 记录启动崩溃次数，达到阈值则清理缓存，重置 App。
   2. 设置独立的安全模式页面，提供联系客服，应用商店更新，杀死重启 App 等操作。

# 全流程 Crash 长效治理

开发阶段

- 统一编码规范，技术评审、CodeReview 机制。
- 架构优化：能力收敛、统一容错。

测试阶段

- 功能测试、自动化测试、回归测试、覆盖安装。
- 特殊场景、机型等边界测试。
- 云测平台。

合码阶段

- 编译检测、静态扫描。
- 预编译流程、主流程自动回归。

发布阶段

- 多轮灰度

上线运维阶段

- 灵敏监控
- 回滚、降级策略
- 热修复
