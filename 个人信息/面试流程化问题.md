# 自我介绍

 17 年软件工程专业本科毕业，目前所在公司是美团。
 
“我在美团主要负责客户端基础技术方向，可以概括为三个层面：

1. **性能与稳定性**：主导启动耗时优化（冷启从 4.3s 降到 3.1s）、包体积卡控与治理、Crash/ANR 问题修复，启动崩溃演练，热修复。
    
2. **监控与告警体系**：从 0 到 1 搭建告警全流程，制定 SOP，把有效告警率提升到接近 100%，问题复盘量下降 80%。
    
3. **工程化与通用基架**：维护集成/打包/发版流水线、迭代壳工程配置、组件化架构；开发三端通用的基架库（用户/骑手/商家）；同时支撑闪屏、首页、定位等业务模块的迭代。

整体上，我希望找一个能继续深入基建方向的团队，比如性能平台、效能工具或通用组件库的建设。”

# 为什么离职
  
  “美团那次变动确实是公司层面的组织调整，整个基础体验团队的方向被重新规划了。我当时的工作内容比较杂，大量时间在处理线上紧急问题和数据排查，虽然解决了问题，但很难沉淀出系统性的技术成果。我希望找一个更注重技术深度和工程建设的团队，这也是为什么我现在特别关注基建方向的岗位。”

## 如果面试官追问“具体什么工作内容不满意？”

“主要是工作模式的问题。大部分时间在响应各种线上紧急问题，比如某个模块崩溃率突增、某个页面加载慢，我需要快速跟进、查日志、跑数据定位原因。这些工作当然有必要，但长期下来我发现自己在做的事情比较碎片化，很少有机会做一个完整的、有深度的技术项目。我希望下一份工作能更注重技术规划和系统性的质量建设，而不是长期处于被动响应状态。”

# 短期经历原因

加入后我发现，团队的需求节奏非常快，排期压力很大，同时项目的技术债务和架构问题也比较突出，导致日常开发大部分时间在应对紧急需求和历史坑，很难做深入的技术建设和质量提升。这和我个人接下来希望追求的方向——在一个重视架构、工程效率和长期健康的团队里持续深耕不太一致。所以经过短暂尝试后，我主动选择了离开，重新寻找更匹配的机会。
