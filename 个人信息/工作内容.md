
## Java Crash 抓取原理：

通过实现 `Thread.UncaughtExceptionHandler` 接口，调用 `Thread.setDefaultUncaughtExceptionHandler()` 全局注册。当主线程或子线程发生未捕获异常时，系统回调 `uncaughtException()` 方法，在回调中通过 `Throwable.getStackTrace()` 收集完整堆栈、线程信息、设备信息等，序列化写入本地文件，在**下次启动时**读取并上报。

## Native Crash 抓取原理：

通过 `sigaction()` 注册自定义 Signal Handler 拦截 Linux 信号，通过 `libunwindstack` 展开调用栈，通过 `dladdr()` 将地址映射为 `.so` 名称和偏移量，写入本地文件下次启动上报，上报后再用 `addr2line` 还原为具体函数和行号。

## ANR 抓取原理：

  - 启动一个监控线程，以固定间隔(通常 5s)向主线程的 `Handler` post 一个心跳任务，任务执行时更新一个时间戳。监控线程每次 sleep 结束后检查时间戳是否被更新，若超过阈值仍未更新，则判定主线程发生阻塞，此时通过`Looper.getMainLooper().getThread().getStackTrace()` 抓取主线程堆栈上报。
  - Android 系统在触发 ANR 时，会向目标进程发送 SIGQUIT，通过 `sigaction()` 提前注册 `SIGQUIT` 的自定义 Handler，收到信号后在 Signal Handler 中抓取所有线程的堆栈信息。

### 主线程消息队列监控

- 方案一：Printer 插桩（无侵入）

```kotlin
object MainLooperMonitor {
    
    private var dispatchStartTime = 0L
    private var currentMsg = ""
    
    fun start() {
        Looper.getMainLooper().setMessageLogging { log ->
            if (log.startsWith(">>>>>")) {
                // 消息开始分发
                dispatchStartTime = SystemClock.elapsedRealtime()
                currentMsg = log
            } else if (log.startsWith("<<<<<")) {
                // 消息分发结束
                val cost = SystemClock.elapsedRealtime() - dispatchStartTime
                if (cost > 100) {
                    // 单条消息耗时超 100ms，记录
                    reportSlowMessage(currentMsg, cost)
                }
            }
        }
    }
    
    private fun reportSlowMessage(msg: String, costMs: Long) {
        // 解析消息来源
        // log 格式：>>>>> Dispatching to Handler{...} xxx: 0
        val handlerInfo = parseHandlerInfo(msg)
        
        report("slow_main_message", mapOf(
            "cost_ms" to costMs,
            "handler" to handlerInfo,
            // 同时抓取主线程堆栈，定位具体执行位置
            "stack" to getMainThreadStack()
        ))
    }
}
```

- 方案二：消息队列深度监控（反射）

```kotlin
object MessageQueueMonitor {
    
    // 反射获取消息队列中的消息数量
    fun getQueueDepth(): Int {
        return try {
            val looper = Looper.getMainLooper()
            val queueField = Looper::class.java.getDeclaredField("mQueue")
            queueField.isAccessible = true
            val queue = queueField.get(looper) as MessageQueue
            
            val msgField = MessageQueue::class.java.getDeclaredField("mMessages")
            msgField.isAccessible = true
            var msg = msgField.get(queue)
            
            // 遍历链表计数
            var count = 0
            val nextField = Message::class.java.getDeclaredField("next")
            nextField.isAccessible = true
            while (msg != null) {
                count++
                msg = nextField.get(msg)
            }
            count
        } catch (e: Exception) {
            -1
        }
    }
    
    // 获取队列中最老消息的等待时间
    fun getOldestMessageWaitTime(): Long {
        return try {
            // 反射获取 mMessages 的 when 字段（消息预期执行时间）
            val queue = getMessageQueue()
            val msgField = MessageQueue::class.java.getDeclaredField("mMessages")
            msgField.isAccessible = true
            val headMsg = msgField.get(queue) ?: return 0
            
            val whenField = Message::class.java.getDeclaredField("when")
            whenField.isAccessible = true
            val expectedTime = whenField.getLong(headMsg)
            
            maxOf(0, SystemClock.uptimeMillis() - expectedTime)
        } catch (e: Exception) {
            -1
        }
    }
}
```

## 启动框架 Aurora

 1.任务分阶段调度

   将启动过程划分为 Application 初始化、Activity 初始化、Activity 加载完成 3 个阶段，也可以自定义时机。

 2.支持主进程和 push 进程、自定义进程。

 3.依赖拓扑排序

   - 通过 beforeTaskNames() 方法声明任务依赖关系，框架自动构建 DAG(有向无环图) 并拓扑排序(线性执行序列，所有任务的依赖，都在该任务之前执行完毕)。
   
   - 支持同步任务（AuroraUITask）和异步任务（AuroraAsyncTask）混合编排。

## 启动防护原理

  - 自动清理：连续崩溃达阈值(默认3次)时清理缓存文件(排除白名单目录）
  - 安全模式：超过阈值后进入安全模式页面，执行定向清理/补丁修复等操作
  - 设备分级：根据设备等级动态调整卡死检测阈值（高端机8s/低端机16s）

## 包体

- dynLoader 动态加载 so

  针对 assets 资源和 so 资源，在编译期把资源上传到服务端，运行时再对资源进行下载和解压。业务方使用时校验 so 版本号和 md5，可用则直接复用。
